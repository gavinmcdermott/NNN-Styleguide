Page leads

Description

Page leads where the main title for the page is held, along with any other primary actions (buttons) that affect content on the page as a whole.

Example

Hannah

Code snippet

We're using some top-level action buttons in this example.

<div class="page-lead lastGroup"> <div class="unit size1of2"> <h1>Hannah</h1> </div> <div class="lastUnit size1of2"> <div class="page-lead__buttons float--right"> <button class="button pri0">Pause deliveries</button> </div> </div> </div>

Sections & subsections

Description

Sections are what we use to wrap blocks of related content in the app. Typically they have an h3 as a lead in for the section. Within the section, if there are different areas of related content, we use the .sub-section tag paired with an h4 title for the content within. This is actually how the styleguide is structured.

There is a 72px bottom margin between .sections.

There is a 48px bottom margin between .sub-sections.

Panels

Description

Panels are typically used to call out specific areas in which the user is to take some kind of action.

- They are hidden by default, and animate into view. Thus you need to add an active class in order for them to show up.

- They use an h3 tag for their header.

Code snippet

<div class="panel active"> <div class="lastUnit size1of1"> <h3 class="">This is a panel header</h3> <p>This is some panel content, and some more content with it.</p> <a class="button pri0">Panel button</a> </div> </div>

Buttons

Description

Buttons come after form elements, and are typically wrapped in a .button-group class which handles padding and spacing.

- Buttons can either be a tags with the .button class applied, or a button tag with the same .button tag applied.

- The colors of the buttons are managed by the themes used in the NNN app.

Example

Save Cancel

Code snippet

<div class="line"> <div class="lastUnit size1of1 button-group"> <a class="button pri0" href="">Save</a> <a class="button pri1" href="">Cancel</a> </div> </div>

Inputs

Text input description

Basic form text inputs used throughout the app

- Input fields are wrapped with the .field-wrap class.

Text input example

Text input code snippet

<div class="field-wrap"> <label>State</label> <input type="text" /> </div>

Increment input description

And incrementing field input used with small incremental values. We don't use them with a lot of values; in which case a select input should be used.

- Note the use of the .field-wrap and .with-buttons classes for this input

- Since it is not an actual text input, we also use a custom .fake-input class in place of the text input normally used.

Increment input example

2 and 3/8 cups per meal

Increment input code snippet

<div class="field-wrap with-button"> <label for="">Choose meal portion</label> <div class="fake-input"> 2 and 3/8 cups per meal <span class="button-group"> <a href="" class="button pri1 fa fa-minus small"></a> <a href="" class="button pri1 fa fa-plus small"></a> </span> </div> </div>

Selects

Description

Selects are used when selecting a bunch of possible options. We use the native element for better access on mobile devices.

Example

Code snippet

<div class="field-wrap"> <label for="">State</label> <div class="select-wrap"> <select> <option value="volvo">CA</option> <option value="saab">NE</option> <option value="mercedes">WI</option> </select> <span class="icon fa fa-chevron-down"></span> </div> </div>

Radios

No-descriptive-text description

Radios are used when choosing between mutually exclusive options in the app.

No-descriptive-text example

No-descriptive-text code snippet

<label class="radio" for="cat"> <input type="radio" value="cat" name="quality" id="cat"> <span>Cat</span> </label>

With-descriptive-text description

This radio can also have a nice little description with it :)

With-descriptive-text example

With-descriptive-text code snippet

<label class="radio" for="dog"> <input type="radio" value="dog" name="quality" id="dog"> <span> <span>Dog</span> <span class="content"> <p class="nomargin">Includes beef, carrots, potatoes, eggs, etc., and a custom vet-formulated nutrient blend and some other things too</p> </span> </span> </label>

Checkboxes

No-descriptive-text description

Checkboxes are pretty self explanatory...

No-descriptive-text example

No-descriptive-text code snippet

<label class="checkbox"> <input type="checkbox" class="option-input" /> <span>This checkbox has no descriptive text</span> </label>

With-descriptive-text description

Checkboxes can also have descriptions.

With-descriptive-text example

With-descriptive-text code snippet

<label class="checkbox"> <input type="checkbox" class="option-input" checked /> <span>I accept you, always, NomNom</span> <p class="description">This is some descriptive text with it</p> </label>

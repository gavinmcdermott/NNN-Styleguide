Vertical rhythm

A note on vertical rhythm

None of the header tags (h1, h2, h3, or h4) have a noticeable margin on them, we use utility classes to handle the vertical rhythm.

As such, here are the standard methods we use to achieve vertical rhythm in the app.

Code snippets

For h2's that occupy an entire horizontal line:

<div class="line"> <div class="lastUnit size1of1 below--54"> <h2 class="f--reg">Some page subtitle</h2> </div> </div>

Note, we can ditch the line wrapper if we don't take up the width.

<div class="lastUnit size1of1 below--54"> <h2 class="f--reg">Some page subtitle</h2> </div>

Vertical spacings used on headers

Here are the font utils we use to separate to keep things flexible.

  • Header Tag
    Class Used
  • page-level h1
    usually in a wrapper that has a .page-lead class
  • h2
    .below--54 class
  • h3
    .below--12 class
  • h4
    no margin class needed

Font utilities

Here are the font utils we use to separate to keep things flexible.

  • CSS Class
    Result
  • .f--small
    makes font size 14px
  • .f--bold
    bolds text weight (fw 900)
  • .f--reg
    regular text weight (fw 400)
  • .f--warn
    sets text color to #EA4343
  • .align--center
    center aligns the text in the element
  • .el--dim
    sets the elements opacity to 75%
